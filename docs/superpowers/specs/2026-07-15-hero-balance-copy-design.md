# 首页首屏左栏平衡与结果导向文案设计

## 范围

仅调整首页首屏左栏内部的垂直节奏、Task Core 中心细节和首屏底部过渡文案。导航、CTA 内容与功能、右侧星图结构、其他页面模块保持不变。

## 设计方案

- 桌面端左栏采用“上下锚定”节奏：身份、姓名、价值主张、技术方向、说明与 CTA 保持紧凑的上部信息组；三项能力证明下沉至左栏底部，形成清晰的证据锚点。
- 不通过放大字号、增加装饰或平均拉开所有间距填充空间；保留暖白、编辑式网格和大面积留白。
- 低高度桌面视口继续使用紧凑规则，确保固定导航加首屏仍为一个视口。
- 移动端保持自然文档流，能力证明仍紧随 CTA，不使用底部锚定。
- 删除 Task Core 中心红点，不改变外壳、金属边缘、陶瓷内核和蚀刻环。
- 首屏底部文案改为：`把复杂问题拆成可执行路径，用工程确定性兑现可验证结果`。

## 验收

- 1440px 桌面端左栏底部不再出现缺乏意图的大片空白，能力证明与首屏底边形成稳定关系。
- 1366px 与低高度桌面视口不发生内容碰撞或裁切。
- 390px 移动端维持原有顺序与可读性，无横向溢出。
- Task Core 中心没有红点。
- 新文案在桌面端保持单行或稳定居中，在移动端不造成溢出。
