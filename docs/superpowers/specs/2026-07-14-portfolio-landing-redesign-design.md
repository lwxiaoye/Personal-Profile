# 梁伟业个人着陆页改造设计

日期：2026-07-14

## 1. 目标

本次改造把 `lwxiaoye.top` 明确为梁伟业的个人介绍与 Agent 项目入口。页面首先服务招聘者和技术面试官，帮助访客在短时间内确认三件事：梁伟业是谁、能独立负责哪些 Agent 应用链路、项目是否有可验证的工程结果。

视觉上延续 `reference.png` 的暖白、黑色和编辑式排版。改造不引入科技蓝大屏、霓虹宇宙背景或与现有页面冲突的产品化组件风格。

## 2. 信息架构与未来路径

站点采用主域名路径式结构：

- `lwxiaoye.top`：个人介绍页
- `lwxiaoye.top/career`：CareerForge-AI
- `lwxiaoye.top/service`：多智能体客服
- `lwxiaoye.top/medical`：医疗 RAG

项目实际部署位置由用户在后续提供。本轮页面改造先建立集中式项目链接配置；地址未提供时不产生可点击的空链接，而是显示明确的“部署地址待提供”状态。

未来若接入单点登录，使用统一认证服务管理身份。项目接口必须保持命名空间隔离：`/api/career/*`、`/api/service/*`、`/api/medical/*`。单点登录负责统一身份，不替代接口隔离、Cookie 范围和回调地址管理。

## 3. 页面标题与首屏文案

浏览器标题使用：

> 梁伟业｜Agent 应用开发

首屏保留姓名作为最大视觉锚点，能力标题改为：

> 把 Agent 做成真正能交付的产品

辅助说明使用：

> Agent 应用开发 / LangGraph / RAG / Harness Engineering

首屏指标继续展示三个真实证据：3 个完整 Agent 项目、意图识别约 95%、RAG 召回率 91.3%。不新增无法从简历或项目中验证的指标。

## 4. 首屏与板块高度

桌面端导航栏与首屏主体合计占满一个视口：

- 导航栏高度：66px
- 首屏主体高度：`calc(100svh - 66px)`
- 首屏的滚动提示和底部说明必须放在首屏内部
- 100% 浏览器缩放时不能露出下一板块

项目、关于我、技术能力、经历和联系板块在默认收起状态下以 `min-height: 100svh` 为目标。项目或技术能力展开后允许板块自然增高，不能为了维持一屏而裁切文字。

小于 820px 的设备取消强制一屏，采用自然高度和连续滚动。移动端优先保证标题、按钮、星图标签和展开内容完整可读。

## 5. 代表项目结构与文案

项目列表一次只展开一个项目。默认状态保持三行紧凑列表；展开状态采用三栏：项目与职责、关键实现、团队贡献与证据。展开后的内容优先控制在常见桌面视口内，但内容完整性高于强制一屏。

### 5.1 CareerForge-AI

项目与职责：

> 我负责 AI 面试官核心模块，也补齐了简历助手的关键链路。目标不是让模型“给建议”，而是让它能读取、修改、撤销和导出真实简历，并围绕目标岗位完成多轮面试。

关键实现：

- 把简历读取、局部更新、写前快照和 PDF 导出封装成受约束工具，由后端 Harness 统一校验。
- 设计简历驱动提问、逐轮追问、6 维评分和报告生成；模型异常时走兜底，不让一次错误中断面试。
- 接入岗位与技术知识检索，让提问和追问有材料依据，而不是只靠通用 Prompt。

团队贡献：

> 能独立接住一条 AI 功能链路，从需求、架构和 Prompt，一直做到接口、前端反馈与异常处理。

### 5.2 多智能体客服

项目说明：

> 我从零搭建工作流，用显式状态图组织分类、路由和回复生成。针对分类结果多行、夹标点和中英混排的问题补了规范化层，把意图识别从约 70% 提升到约 95%。

重点呈现 Agent 架构、稳定性治理、越界意图短路、双层会话管理和 SSE 流式交互。团队贡献落在“能设计可读、可调试、可增量扩展的 Agent 工作流，并处理真实生产边界”。

### 5.3 医疗 RAG

项目说明：

> 我完成数据入库、Milvus 混合检索、RRF 融合、检索质量评估和事实核验。召回率达到 91.3%，精确率达到 92.1%，失败时会自动补充搜索并回退重试。

重点呈现 RAG 全链路、Token 预算控制、对话摘要压缩、主动追问、并行子查询和 FastAPI 后端基础设施。团队贡献落在“能把检索、评测、回退和业务接口组成完整闭环”。

### 5.4 写作规则

文案使用第一人称、具体动作、真实问题和已有数字。避免“赋能、打造、深度融合、全方位提升”等无法验证的表达。每段回答一个明确问题，不堆叠同义形容词，不为了显得完整而使用机械的三段式总结。

## 6. 3D Agent 运行星图

采用已确认的“编辑式机械装置”方案：

- 三层细线轨道分别沿不同空间轴缓慢旋转
- 任务核心做小幅悬浮，形成前后景深
- Intent、Plan、RAG、Tools、Verify 按顺序依次激活
- 活动轨迹与左侧鼠标洞察遮罩的当前阶段同步
- 当前节点使用一处状态色，其余保持暖白、黑色和灰色
- 不使用大面积渐变、发光球体、星空背景或重度镜头动画

实现优先使用现有 Canvas 绘制能力配合 CSS 3D 变换，不为了效果引入重型 3D 场景依赖。动画持续时间保持缓慢，鼠标悬停节点时可锁定当前阶段；鼠标离开后恢复自动轮转。

当系统开启 `prefers-reduced-motion: reduce` 时，停止轨道旋转、节点自动轮转和悬浮，只保留清晰的静态空间构图。移动端减少轨道层数和阴影，确保文字标签不重叠。

## 7. 交互规则

- 顶部导航仍在同一页面内滚动
- “查看项目”主按钮滚动到代表项目板块
- 项目行点击后在当前页面展开，一次只打开一项
- 项目详情最终提供“在线体验”和“查看源码”两个出口
- 左侧洞察遮罩继续被裁切在左半首屏内，并与星图阶段同步
- 键盘用户可以操作导航、项目行、技术能力行和星图节点

## 8. 验证与测试

实现完成后需要验证：

- 1440×900、1920×1080 和 1366×768 下，导航栏加首屏严格等于一个视口，不露出下一板块
- 项目默认状态、展开状态和一次只展开一项
- 五个星图节点的自动轮转、悬停锁定、点击锁定和洞察遮罩同步
- `prefers-reduced-motion` 下无持续动画
- 820px 以下移动布局无文字裁切、节点重叠和横向滚动
- 浏览器标题、GitHub 链接、简历下载和联系信息正确
- 项目部署地址缺失时不存在空链接或误导性可点击按钮

- Cookie、Token、登录回调和跨项目退出策略
- 三个 Agent 项目本身的登录页和内部页面改造
