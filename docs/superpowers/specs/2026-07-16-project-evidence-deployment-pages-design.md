# 项目证据与部署状态页设计规范

## 目标

在不改变个人主页既有暖白、黑色编辑排版、细线与克制暖红信号的前提下，完成三项调整：

1. 将 CareerForge-AI 源码地址统一为 `https://github.com/lwxiaoye/CareerForge-AI`。
2. 删除缺少测试集说明的“约 70% 提升到约 95%”数据，改成可解释的工程结果与一条真实、匿名的使用反馈。
3. 为尚未完成正式部署的多智能体客服和医疗 RAG 提供真实可访问的 `/service/`、`/medical/` 状态页，让“前往体验”不再伪装成在线应用，也不再跳转到源码仓库。

## 已确认的视觉方向

采用用户选择的方案 A“主站首屏缩影”。两个部署状态页必须复用主站的视觉语言：

- 暖白背景、黑色主文字、黑色实心主按钮、灰色说明文字。
- 细线分隔、少量暖红 `#e54e37` 状态信号，不引入新的品牌色。
- 顶部显示 `LIANG WEIYE` 与“返回作品集”。
- 左侧放项目状态、项目名称、部署说明和操作按钮；右侧展示现有三轨道 Agent 图标。
- 底部显示项目技术栈与 `COMING SOON` 状态。
- 桌面为左右双栏，移动端自然堆叠，不产生横向滚动。

三轨道图标继续使用 `public/agent-orbit-favicon-master.png`，不重新生成或替换图形。

## CareerForge-AI 证据调整

### 源码归属

主页 CareerForge-AI 的源码链接必须统一为 `https://github.com/lwxiaoye/CareerForge-AI`，页面中不得再出现 `Dloading666/CareerForge-AI`。

### 删除无依据指标

全站删除 `约 70% 提升到约 95%`、`约 95% 意图识别` 以及任何等价的意图识别百分比结论。

多智能体客服的工程结果统一表达为：

`完成“分类—路由—专业 Agent—SSE 回传”闭环，并通过规范化层处理多行、夹标点和中英文混排导致的路由偏差。`

该表述可用于项目详情、技术能力证据和经历区，但应按各模块语境缩短，不能重新添加未经验证的数字。

### 真实使用反馈

CareerForge-AI 项目详情增加“真实使用反馈”，使用匿名、克制的第三人称描述：

> 一位同学的原简历因表格化布局、信息缺失和重点不清，被辅导员指出存在较大问题。使用 CareerForge 补充关键信息并重新组织内容后，他反馈填写过程方便，生成的简历更加完整、重点更清晰；随后通过辅导员内部推荐进入公司实习。

文案必须保留“随后”这一时间关系，不能改写成“因此获得实习”或暗示 CareerForge 是进入实习的唯一原因。不得添加公司名称、薪资、面试通过率或用户数量。

## 组件与路由结构

- 在 `src/portfolio-data.js` 中集中维护项目体验地址、源码地址、部署页文案和真实反馈。
- 新建独立的部署状态页组件，接收项目数据并渲染同一套布局。
- `src/App.jsx` 根据规范化后的 `window.location.pathname` 渲染主页、`/service/` 或 `/medical/`；不引入额外路由依赖。
- 状态页必须复用现有字体、颜色变量和按钮风格，不复制整套主页组件。

## 可访问性与降级

- 页面必须有唯一的 `h1`，项目状态、说明和按钮保持合理阅读顺序。
- 三轨道图标使用明确的替代文本。
- “查看源码”和“返回作品集”必须有清晰的可访问名称与可见焦点。
- 小屏幕不依赖悬停；`prefers-reduced-motion` 下不新增持续动画。
- 状态信息不能只靠暖红色表达，必须同时出现“正在部署中”文字。

## 验证标准

- 自动化测试先验证新行为并在实现前失败。
- `/career/`、`/service/`、`/medical/` 的主页入口目标正确。
- 两个状态页显示对应项目名称、部署说明、技术栈、源码和返回主页操作。
- CareerForge 源码只指向 `lwxiaoye/CareerForge-AI`。
- 全站测试确认不再出现 `70%`、`95%` 或旧 CareerForge 仓库地址。
- CareerForge 项目详情显示经过确认的真实使用反馈。
- 生产构建通过；桌面和 390px 移动端均无横向溢出或控制台错误。

## 不在本次范围

- 不部署多智能体客服或医疗 RAG 的实际后端服务。
- 不虚构部署百分比、发布日期或用户结果数字。
- 不修改 CareerForge 的 `/career/` 反向代理和现有服务。
- 不重新设计个人主页其他模块。
