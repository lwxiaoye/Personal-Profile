# Agent 主题图标与项目临时体验链接设计

## 目标

用符合个人作品集视觉语言的新图标替换当前浏览器标签页图标，并在尚未部署的“多智能体客服”和“医疗 RAG”项目中同时提供“查看源码”与“前往体验”入口。

## 浏览器图标

- 采用“Agent 轨道标记”方向。
- 用户指定的三轨道附件是视觉源真值，并优先于此前的文字方案；其仓库母图使用暖白色正方形背景、黑色精密核心和三条黑色轨道。
- 保持且只使用母图中的一个以 `#e54e37` 为目标色的暖红运行信号，呼应首页 Agent 运行时星图。
- 保留选定母图本身已有的渐变、明暗与材质表现，不额外添加视觉效果；图标保持无文字。
- 直接采用已存入仓库的选定高分辨率母图，制作适合浏览器标签页的 32×32 与 16×16 PNG 资源，不重新生成图标。
- HTML 显式声明 favicon，保证开发和生产构建均加载新资源。

## 项目链接

### CareerForge-AI

- “前往体验”继续指向同域路径 `/career/`，在当前标签页打开。
- “查看源码”保持现有仓库地址和新标签页行为。

### 多智能体客服

- “查看源码”指向 `https://github.com/lwxiaoye/Agent-`。
- 在项目部署完成前，“前往体验”临时指向同一仓库。
- 两个 GitHub 入口都在新标签页打开，并带安全的 `rel` 属性。

### 医疗 RAG

- “查看源码”指向 `https://github.com/lwxiaoye/medical-RAG-`。
- 在项目部署完成前，“前往体验”临时指向同一仓库。
- 两个 GitHub 入口都在新标签页打开，并带安全的 `rel` 属性。

## 交互与可访问性

- 项目卡片继续使用现有中文按钮文案和展开方式。
- 每个链接保留包含项目名称的无障碍名称。
- 仅外部 GitHub 链接新开标签页；同域 `/career/` 不改变当前导航体验。

## 验证

- 测试三个项目的“前往体验”和“查看源码”地址、外部链接属性及 CareerForge 同域行为。
- 验证 favicon 资源存在、尺寸正确，并能被生产构建引用。
- 运行自动化测试和生产构建。
- 在桌面与移动视口检查项目卡片链接，并在浏览器中确认新 favicon。
